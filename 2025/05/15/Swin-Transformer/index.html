Swin_Transformer

1.Patch Partition + Linear Embedding = Patch Embed

在Swin Transformer网络整体框架中，Patch Partition与Linear Embedding模块在代码实现中，是作为一个整体在PatchEmbed类中来实现的。

从数据的尺寸角度来看，输入为[H, W, 3]，输出为[H/4, W/4, C] (在，Swin-T中C=96)。则通过一个卷积层，输入维度是3，输出维度是96，卷积核大小为(patch_size, patch_size)=(4, 4)，步长为(4, 4)，一步操作就可达到目的，即代码中的self.proj操作。

以[B, 3, 224, 224]的输入图像为例，经过这一步后得到[B, 96, 56, 56]的数据。然后展平为[B, 96, 3136]，交换位置[B, 3136, 96]。最后经过norm_layer, 即得到96个token，每个token的维度是3136. 返回[B, 3136, 96], 56, 56.

2.Swin Transformer Block

[B, 3, 224, 224]的输入数据经过第一步的Patch Embed之后，变为[B, 3136, 96], H=56, W=56，然后流入swin transformer block，在这一步不会改变数据的尺寸，所以输出数据的维度还是[B, 3136, 96].

Swin Transformer Block中包含WindowAttention模块（line30）和MLP模块（line37）.分别在2.1和2.2详解。

Swin Transformer Block通过W-MSA和SW-MSA，残差连接，前馈网络。实现了特征提取的功能。

第46行，将输入的[B, 3136, 96]的数据view为[B, 56, 56, 96].

第50-54行，把feature map给pad到window size的整数倍, x = F.pad(x, (0, 0, pad_l, pad_r, pad_t, pad_b)) ，x是四维张量[B,H,W,C]=[B,56,56,96] ，(0,0, pad_l, pad_r, pad_t, pad_b)表示从最后一个维度C开始，按照C，W，H，B的顺序依次填充，0,0表示在C维度上不填充，pad_l, pad_r表示在W维度上左边填充pad_l，右边填充pad_r，pad_t, pad_b表示在H维度上，上面填充pad_t，下面填充pad_b，B维度一般不填充。由于56是7的整数倍，所以pad为0，数据尺寸还是[B,56,56,96]。

第58行，shifted_x = torch.roll(x, shifts=(-self.shift_size, -self.shift_size), dims=(1, 2))，self.shift_size=window_size // 2 = 7 // 2 = 3，dim=1表示H（高度）维度，dim=2表示W（宽度）维度。torch.roll用于沿指定维度对张量进行位移，shifts中的负值表示向上，向左方向位置，故，这行代码表示feature map整体向左上方向移动了3个像素。被移出边界的数据会重新出现在另外一侧，左边的数据会出现在右边，上面的数据会出现在下面。所以数据的尺寸还是[B, Hp, Wp, C]=[B,56,56,96]

以下图移动距离为1的情况为例：

第64行，window partition，将feature map按照window_size划分成一个个没有重叠的window，然后对数据维度做变换，得到[B*num_windows, Mh, Mw, C]=[B*64,7,7,96]，Mh和Mw是一个window中的高和宽。然后再变换为[B*num_windows, Mh*Mw, C]=[B*64,49,96].

第68行，将数据[B*64,49,96]，和attn_mask，输入到WindowAttention计算每个Window的自注意力，这一步数据的尺寸不改变，还是[B*64, 49, 96].

第71-72行，合并窗口，将数据变为[B*64, 7, 7, 96]，再变为[B, 56, 56, 96]。

第75-78行，平移窗口的逆操作，将数据复原，尺寸还是[B, 56, 56, 96]。

第80-82行，去掉前面加的pad，由于之前pad为0，所以尺寸还是[B, 56, 56, 96]。

第84行，将数据view回[B, 3136, 96].

第87行，将x经过drop path层后再加上其残差连接，得到新的x。

第88行，将新的x经过LN层和MLP层，再加上其残差连接，得到输出。

Swin transformer block主要做的事情是：复制x用于残差连接，将数据还原成feature map，然后pad成window size的整数倍，

2.1 WindowAttention

这个模块用于实现W-MSA功能和SW-MSA功能，即在给定的窗口中计算多头自注意力，值得注意的是，SW-MSA中的shift功能已经在swin transformer block中实现了，故此模块没有该部分的实现，这个模块主要实现了relative postition bias(相对位置编码)。

第23-39行，实现了relative position bias，用于在自注意力中加入位置信息。

第24行，创建一个可学习的参数表，来存储relative position bias，(2 * window_size[0] - 1) * (2 * window_size[1] - 1)是相对位置的范围，以上图为例，window_size=2，则在高度H方向上能取到的值是{-1，0，1}，也就是2 2 - 1 = 3个数，宽度W方向上同理。由于采用多头注意力机制，所以每个head上都会有一个大小为`[2window_size - 1, 2window_size - 1]的relative position bias table，即self.relative_position_bias_table的尺寸为[2window_size - 1, 2*window_size - 1, num_heads]`.

第28-30行，为窗口内的每个token生成二维坐标（绝对位置索引），coords_h用来生成高度H方向上的坐标，coords_w用来生成宽度W方向上的坐标，其范围是{0, 1, 2, …, window_size-1}，以上图中左下角22的矩阵为例，在高度H和宽度W方向上的取值都是0 到 window_size-1(2 - 1 = 1)。torch.meshgrid接收一个一维的张量（通常是坐标轴的范围），返回对应的高维网格。[coords_h, coords_w]是两个一维张量，表示高度和宽度的坐标，则会*返回两个对应的高维网格，indexing=‘ij’表示高维网格的索引方式为行优先，即coord_h对应行，coords_w对应列。举例说明：

得到2个shape=[Mh, Mw]的坐标后，将其按指定维度堆叠，生成一个新的张量coords，shape=[2, Mh, Mw].其中coords[0, :, :]表示网格中所有位置的高度坐标，coords[1, :, :]表示网格中所有位置的宽度坐标。

这行代码的最终目的是为窗口内的每个 token 生成一个二维坐标网格，存储在 coords 中，形状为 [2, Mh, Mw]。这些坐标随后用于计算 token 之间的相对位置 (Δh, Δw)，以便为自注意力机制引入相对位置偏置。

第31行，将二维坐标展平到一维，shape从[2, Mh, Mw]变为[2, Mh*Mw]，以便计算每个token的相对位置。

第33行，通过广播和减法操作，计算窗口内每个token的相对位置(Δh, Δw)，即一个 token 相对于另一个 token 在高度和宽度上的坐标差。举例说明：

第34行，relative_coords.permute(1, 2, 0)：将形状从 [2, Mh*Mw, Mh*Mw] 转换为 [Mh*Mw, Mh*Mw, 2]，最后一维表示 (Δh, Δw)。contiguous()：确保内存连续性。

第35行，将高度方向的相对位置 Δh（范围 [-Mh+1, Mh-1]）平移到 [0, 2*Mh-2]。

第36行，将宽度方向的相对位置 Δh（范围 [-Mh+1, Mh-1]）平移到 [0, 2*Mh-2]。

第37行，relative_coords[:, :, 0]表示relative_coords的高度差Δh，这行代码将每个Δh值乘以2*Mw-1.

第38行，Δh * (2*Mw-1) + Δw ，计算唯一的索引，形状为 [Mh*Mw, Mh*Mw]，最终的 relative_position_index 是一个索引矩阵，每个元素对应一个相对位置的唯一 ID，用于从 relative_position_bias_table 中查找偏置值。

具体来说，Swin Transformer 使用线性索引公式：index = Δh * (2*Mw-1) + Δw，其中 Δh 和 Δw 都是平移后的非负值。

第39行，将 relative_position_index 注册为模型的缓冲区（buffer），而不是可学习的参数。缓冲区不会被优化器更新，但会随模型保存和加载，确保相对位置索引在推理时保持一致。

其中 B 是通过 relative_position_index 索引得到的相对位置偏置。

第41行，将输入 token 特征通过一个线性变换同时生成 Q、K、V，用于后续的自注意力计算。

第46行，使用截断正态分布（truncated normal distribution）初始化相对位置偏置表 self.relative_position_bias_table。

第70-73行，self.relative_position_bias_table：形状为[(2*Mh-1)*(2*Mw-1), nH]，存储所有可能的相对位置偏置，nH 是注意力头数。 self.relative_position_index：形状为 [Mh*Mw, Mh*Mw]，包含每对 token 的相对位置索引。self.relative_position_index.view(-1)：将索引展平为[Mh*Mw*Mh*Mw]，用于从偏置表中提取对应的偏置值。self.relative_position_bias_table[...].view(Mh*Mw, Mh*Mw, -1)：提取偏置后，重塑为 [Mh*Mw, Mh*Mw, nH]，表示每个 token 对在每个注意力头上的偏置。

attn + relative_position_bias.unsqueeze(0)：将相对位置偏置加到注意力分数上，增强模型对 token 间空间关系的建模。

2.2 MLP

Swin Transformer的MLP模块与ViT的MLP模块完全相同，是一个简单的两层前馈神经网络（FFN），用于在自注意力机制后进一步处理 token 特征，增强模型的非线性表达能力，其结构是两层全连接网络（fc1 → 激活 → Dropout → fc2 → Dropout）。输入和输出通常具有相同的维度（in_features = out_features）

以Swin-T的第一个stage的第一个swin transformer block中的MLP为例，输入数据是[B, 56*56, 96]。

第一个全连接层 (fc1)

  • 操作：self.fc1 = nn.Linear(in_features=96, hidden_features=384)。

  • 形状变化：[B, 3136, 96] → [B, 3136, 384]

    每个 token 的特征从 96 维映射到 384 维，token 数量不变。

之后通过激活函数和dropout，随机丢弃部分值，但不改变张量形状。

第二个全连接层 (fc2)

  • 操作：self.fc1 = nn.Linear(in_features=384, hidden_features=96)。

  • 形状变化：[B, 3136, 384] → [B, 3136, 96]

    每个 token 的特征从 384 维映射到 96维，token 数量不变。

之后通过dropout随机丢弃部分值，不改变张量形状。

3.Patch Merging

在Swin Transformer的网络结构图中，共有4个stage，第一个stage是Linear Embedding + swin transformer block，剩下三个stage都是patch merging + swin transformer block，而在代码的实现中，前三个basic block = swin transformer block + patch merging，最后一个basic block = swin transformer block，与图中有所不同。

第12行，self.reduction是一个线性层，用于将4C的通道数变换为2C，即上图中的Linear步骤。x0, x1, x2, x3表示上图中蓝，橙，绿，红四个。

当输入维度是[B, H*W, C]时，经过PatchMerging，输出维度是[B, (H/2)*(W*2), 2C].

4.Basic Layer（创建mask掩码）

Basic Layer将Swin Transformer Block 和 Patch Merging组合起来，构建出Swin Transformer的4个stage。另一个主要功能是为SW-MSA的计算创建mask掩码。

第25行，self.shift_size = window_size // 2，self.shift_size表示将窗口向右和向下偏移的距离，其值=window_size/2下取整。

第33行，shift_size=0 if (i % 2 == 0) else self.shift_size，对于transformer block而言，是依次堆叠两个block，其中第一个是W-MSA，第二个是SW-MSA，所以表示窗口滑动的距离在0和self.shift_size之间切换。

第48-72行，为SW-MSA创建掩码。
