Swin_Transformer

1.Patch Partition + Linear Embedding = Patch Embed

在Swin Transformer网络整体框架中，Patch Partition与Linear Embedding模块在代码实现中，是作为一个整体在PatchEmbed类中来实现的。从数据的尺寸角度来看，输入为[H, W, 3]，输出为[H/4, W/4, C] (在，Swin-T中C=96)。则通过一个卷积层，输入维度是3，输出维度是96，卷积核大小为(patch_size, patch_size)=(4, 4)，步长为(4, 4)，一步操作就可达到目的，即代码中的self.proj操作。

以[B, 3, 224, 224]的输入图像为例，经过这一步后得到[B, 96, 56, 56]的数据。然后展平为[B, 96, 3136]，最后交换位置[B, 3136, 96]。即96个token，每个token的维度是3136. 返回[B, 3136, 96], 56, 56.

2.Patch Merging

在Swin Transformer的网络结构图中，共有4个stage，第一个stage是Linear Embedding + swin transformer block，剩下三个stage都是patch merging + swin transformer block，而在代码的实现中，前三个basic block = swin transformer block + patch merging，最后一个basic block = swin transformer block，与图中有所不同。

第12行，self.reduction是一个线性层，用于将4C的通道数变换为2C，即上图中的Linear步骤。x0, x1, x2, x3表示上图中蓝，橙，绿，红四个。

当输入维度是[B, H*W, C]时，经过PatchMerging，输出维度是[B, (H/2)*(W*2), 2C].

4.Basic Layer

Basic Layer用于构建Swin Transformer的4个stage。

第25行，self.shift_size = window_size // 2，self.shift_size表示将窗口向右和向下偏移的距离，其值=window_size/2下取整。

第33行，shift_size=0 if (i % 2 == 0) else self.shift_size，对于transformer block而言，是依次堆叠两个block，其中第一个是W-MSA，第二个是SW-MSA，所以表示窗口滑动的距离在0和self.shift_size之间切换。
