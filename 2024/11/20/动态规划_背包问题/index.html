1.01背包

一、动态规划问题的思考方式

f(i, j)表示集合的某种属性，集合中是所有的选法，但属性是一个值，f(i, j)最终会落实到这个值上面。在01背包中，f(i, j)的值表示所有选法的价值的最大值。

所有的动态规划问题都可以用上图的思考方式，分别考虑状态表示是什么？状态计算是什么？状态表示是哪个集合的哪种属性？集合是哪些东西的集合？集合中的元素要满足什么条件？状态计算表示集合的划分，当前的选法可以分成哪些类？怎么把这些类算出来？

对于01背包问题:

集合是所有“从前i个物品中选的总体积不大于j的选法”的集合；

集合中的元素都满足只从前i个物品中挑选，且总体积不大于j；

状态表示是这个集合中所有选法的价值的最大值；

这些集合可以分成两大类：(a)从第1到第i个物品中选，不包含第i个物品的选法的集合。(b)从第1到第i个物品中选，包含第i个物品的选法的集合；

不包含第i个物品的所有选法的价值的最大值 = f(i-1, j); 包含第i个物品的所有选法的价值的最大值 = 去掉第i个物品后，所有选法的价值的最大值 + 第i个物品的价值w[i] = f(i-1, j-v[i])+w[i]; f(i, j) = max(f(i-1, j) ,f(i-1, j-v[i])+w[i] )。

优化的方式：通过恒等变形将i删掉，从而把二维优化到一维。

<1> 删掉i, f[N][N]变为f[N]; 第19行f[i][j] = f[i-1][j]变为f[j] = f[j] ，此步为恒等变形，由于在第i-1层，已经算出了背包容量最大为j时候的f[j]，故在i层时，二维写法中首先继承i-1层的f[j]，即第i层的f[j]保持i-1层的f[j]值不变，故可以直接删掉i。又因为删掉后变为f[j] = f[j]是一个恒等式，所以这行可以直接删掉；

答案是：不行；由于此时j从0到m开始递增遍历，所以j - v[i] < j，故此时f[j - v[i]]中的值已经在第i层中计算过了，这里的f[j - v[i]]表示的其实是f[i][j - v[i]]而不是f[i - 1][j - v[i]]。

所以，我们让j从m到0进行递减的遍历，由于j - v[i] < j，故当计算这一步时，f[j - v[i]]的值还没有被第i层更新，仍然保留的是第i-1层的值，即f[i - 1][j - v[i]] ， 此时我们删去i维，就是恒等变形。

四、多重背包

思考方式

朴素写法

多重背包问题的朴素写法和完全背包问题基本一致，唯一要考虑的是遍历第i个物品的个数时，要考虑s[i]的限制条件。

多重背包的优化

首先考虑类似完全背包的优化方式：

可以看到，f[i][j-v]除了含有和f[i][j]相似的部分之外，还多出了一项f[i-1][j-(s+1)v]+(s+1)w ,而我们无法通过前n项的最大值和第n项的值来求出前n-1项的最大值，故此法行不通。

二进制的优化方式

假设第i个物品有1023个，将第i个物品打包成10组，第一组有1个，第二组有2个，第三组有4个，…，第十组有512个，每组的物品只能选一次，将每个打包起来的物品看作是01背包里面的一个物品，则通过这10组物品，可以拼凑出从0到1023中的任何一个数；

一般的，如果一个物品有s个，将s打包成1，2，4，8，…，$2^k$，c；其中c严格小于$2^{k+1}$。这k+1个打包起来的物品可以表示0到s中的任何一个数，且每个物品最多只能选一次。

例如一个物品有200个，则可以打包成1，2，4，8，16，32，64，73；其中第一块有1个，可以表示0到1，前两块可以表示0到3，前三块可以表示0到7，以此类推，前7块可以表示0到127，全部的物品可以表示0到200；

通过二进制的优化方式，可以将时间复杂度从O(NVS)优化为O(NVlogS)，其中N是物品的种类，V的背包体积，S是物品的个数。

五、分组背包

思考方式

区别在于：完全背包和多重背包是枚举第i个物品选几个，分组背包是枚举第i组物品选哪个。 (热狗划分法)

主要思路和01背包大致相同：

第一层循环遍历物品组数；

第二层循环遍历背包容量；

第三层循环遍历所有的决策；有s+1种决策，s表示该组物品的个数，要么一个都不选，要么选第1个，要么选第2个，……，要么选第s个；

这里与多重背包的类似点：多重背包是每个物品有s个，也是有s+1种决策，要么选0个，要么选1个，要么选2个，……，要么选s个；

多重背包问题是分组背包问题的特殊情况，由于第i个物品最多可以选s个，我们将这s个物品打包起来，第1种是一个都不打包，第2种是打包一个，第3种是打包2个，……，第s+1种是打包s个。将这s种打包看作s个不同的物品（第1种啥都没有）这s个物品只能选一个或者不选；因此可以利用多重背包问题的特殊性做一些优化，如二进制优化，单调队列优化，而分组背包问题只能用for循环遍历来做。
